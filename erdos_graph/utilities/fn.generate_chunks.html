pub fn generate_chunks(
    start: DateTime<Utc>,

Expand description

Divides a date range into smaller chunks of a specified size.

This is useful for breaking large scraping jobs into manageable pieces, allowing for checkpointing and avoiding memory issues with large result sets.

§Arguments

  • start - The beginning of the overall date range
  • end - The end of the overall date range
  • chunk_size - The maximum duration of each chunk

§Returns

A vector of (start, end) date pairs representing sequential chunks. The last chunk may be smaller than chunk_size if the range doesn’t divide evenly.

§Example

use chrono::{Duration, Utc};
use erdos_graph::utilities::generate_chunks;

let start = Utc::now();
let end = start + Duration::days(30);
let chunks = generate_chunks(start, end, Duration::days(7));
assert_eq!(chunks.len(), 5); // 30 days / 7 days per chunk = 5 chunks
