Asynchronously scrape publication records from ArXiv for a given date range.

This is the high-level async entry point that uses ArxivConfig::default() to obtain configuration (which may read environment variables). It fetches pages from the ArXiv Atom API and parses <entry> elements in a streaming, sliding-window manner so memory usage stays proportional to the largest in-flight entry rather than the whole feed.

Parameters

  • start_date, end_date: the inclusive/exclusive time window to include publications (entries with published in [start_date, end_date) are returned).

Returns

  • Ok(Vec<PublicationRecord>) on success with the matched records.

  • Err(...) on network, parsing, or runtime errors.

  • Err(...) on network, parsing, or runtime errors.

Notes

  • For tests or deterministic runs prefer scrape_range_with_config_async and pass a custom ArxivSourceConfig (e.g., with base_url pointing at a mock server).
